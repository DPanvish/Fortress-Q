Slide: 2. Abstract
 Brief overview: Fortress-Q is a comprehensive system demonstrating the impact of quantum computing on blockchain security and providing a quantum-resistant solution.
 Main problem being solved: Vulnerability of legacy cryptographic algorithms (RSA, ECDSA) to Shor's algorithm, which threatens the security of current blockchain assets.
 Key solution approach: Implementing a "Quantum Vault" using Post-Quantum Cryptography (Lattice-based CRYSTALS-Dilithium), simulating Quantum Key Distribution (BB84), and demonstrating quantum mining advantages (Grover's Algorithm).

Slide: 4. Introduction
 Problem statement: Quantum computers utilizing Shor's algorithm can efficiently factor large integers, breaking the security of RSA and ECDSA keys used in cryptocurrencies.
 Objectives: To simulate quantum attacks on legacy wallets and develop a secure, quantum-resistant vault for asset storage.
 Goals: 
  - Demonstrate Shor's algorithm factoring.
  - Implement Lattice-based signatures (Dilithium).
  - Simulate Quantum Key Distribution (BB84).
  - Demonstrate Quantum Mining (Grover's).

Slide: 5. Literature Survey/Review
 Summary of existing solutions: Current blockchains rely on classical ECC/RSA. NIST is standardizing Post-Quantum Cryptography (PQC) algorithms like Kyber and Dilithium.
 What others have done: Theoretical research on PQC and some initial blockchain integrations, but few comprehensive educational/demonstration platforms.
 Gaps or limitations in existing work: Lack of accessible tools to visualize and understand the quantum threat and the migration path to quantum-safe wallets.
 How your project fills these gaps: Provides a practical, interactive simulation of both the attack (Shor's) and the defense (Lattice, QKD) within a unified platform.

Slide: 6. System Analysis
 Proposed system overview: A hybrid system combining a Python-based Quantum Engine (using Qiskit) with a Node.js backend and a React frontend.
 Software requirements: 
  - Python 3.x (Qiskit, NumPy)
  - Node.js (Express)
  - React (Frontend)
  - Hardhat (Ethereum Simulation)
 Hardware requirements: Standard PC (Quantum behavior is simulated via Qiskit BasicProvider).
